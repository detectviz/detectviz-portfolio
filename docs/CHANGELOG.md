# 變更日誌

本文檔記錄 Portfolio 專案的所有重要變更，遵循 [Keep a Changelog](https://keepachangelog.com/) 格式。

## 版本說明

版本號格式：主版本.次版本.修訂版（MAJOR.MINOR.PATCH）

- **主版本**：重大架構變更或不相容的 API 變更
- **次版本**：新增功能，向後相容
- **修訂版**：Bug 修復與小幅改進

## 貢獻指南

變更日誌遵循以下原則：

1. **版本號規範**：遵循語義化版本（Semantic Versioning）
2. **分類標準**：
   - **Added**：新增功能
   - **Changed**：既有功能的變更
   - **Deprecated**：即將移除的功能
   - **Removed**：已移除的功能
   - **Fixed**：Bug 修復
   - **Security**：安全性修復
   - **Technical**：技術細節與 Commits
3. **撰寫規範**：
   - 使用現在式動詞（新增、修改、修復）
   - 描述使用者可感知的變更
   - 包含相關 Commit hash（如適用）
   - 提供足夠的上下文資訊
4. **更新時機**：
   - 每次發布新版本時更新
   - 重大功能完成時記錄
   - Bug 修復後記錄
